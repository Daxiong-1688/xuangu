# Stock Selection Skill

## 1. 定位

本 Skill 用于生成“A股策略观察股票池”，属于研究辅助流程，不是面向用户的投资推荐产品，不输出交易指令、目标价、收益保证或确定性方向结论。

## 2. 输入

每日输入为一个结构化市场快照，至少包含：

- 运行日期与实际行情日期；
- 市场指数与整体风险等级；
- 热点板块、产业催化与新闻摘要；
- 板块资金或可替代的标准化资金分；
- 候选股票代码、名称、所属方向；
- 真实收盘价、涨跌幅、20日均线、60日均线；
- 可选的52周高点、成交量变化、成长数据；
- 数据来源、mock 标识与字段完整性说明。

不得用成交额、市值、PE、PB、PS 等字段冒充趋势涨跌幅或均线字段。

## 3. 数据完整性门槛

正式技术结构观察池要求候选样本中的以下字段覆盖率均达到 `config/strategy.yaml` 的门槛：

- `close`
- `pct_change_20`
- `ma20`
- `ma60`

若未达到门槛：

- 保留可用数据；
- 输出 `candidate_observation_pool`；
- 明确说明数据缺口；
- 不称为正式技术 TOP 5；
- 不通过其他指标猜测缺失趋势。

## 4. 评分模型

总分由四部分组成：

1. 热点板块分：热点新鲜度、产业催化、来源质量、是否出现退潮或过热。
2. 资金流分：标准化资金强弱；缺失时使用中性分并披露。
3. 技术结构分：真实涨跌幅、收盘价与20/60日均线关系、52周位置、量价变化。
4. 风险控制分：使用 `100 - risk_score`，风险越高，对总分贡献越低。

正式权重、最低总分、最低技术分和最高风险分以 `config/strategy.yaml` 为准。

## 5. 风险过滤

以下标的不得进入正式观察池：

- ST、退市整理、北交所标的（除非未来人工修改策略范围）；
- 无有效代码、名称或观察价；
- 风险分超过正式策略上限；
- 核心趋势字段覆盖不足时，不得伪装成正式排名；
- 短期涨幅偏快、接近前高或热点退潮信号明显时，应降低排序并写入风险。

## 6. 输出

核心输出为 `selected_stocks.csv`，至少包含：

- 日期、代码、名称、方向、观察价；
- 策略版本、综合分及四类分项；
- 理由摘要、观察周期、风险等级；
- 信号列表、观察池状态、数据质量说明。

同时生成逐股理由与风险分析。措辞统一使用“观察股票池”“策略观察”“趋势延续”“回调观察”“风险提示”等。

## 7. 边界

- 不因单日热点直接形成绝对结论。
- 不自动修改正式策略。
- 不隐藏 mock 数据身份。
- 不把历史表现解释为未来收益保证。
