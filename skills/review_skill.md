# Review Skill

## 1. 定位

本 Skill 对历史“观察股票池”进行事后复盘，目标是识别策略在哪些市场环境、观察周期和信号组合下有效或失效。

## 2. 复盘对象

每次从 `runs/YYYY-MM-DD/selected_stocks.csv` 读取当日观察标的，并从后续日期的 `market_data.json` 读取同一代码的价格快照。

观察周期为：

- 1个交易日；
- 3个交易日；
- 5个交易日；
- 10个交易日。

若后续样本尚未到期，状态为 `pending`；若到期但缺少价格，状态为 `price_unavailable`，二者都不得计为成功或失败。

## 3. 单标的指标

对每只股票、每个观察周期计算：

- 周期收益率；
- 观察窗口内最大涨幅；
- 从窗口历史峰值计算的最大回撤；
- 同期基准指数收益率；
- 同期所属板块收益率；
- 是否跑赢指数；
- 是否跑赢所属板块；
- 是否触发策略风险回撤条件；
- 推荐理由是否成立。

“理由是否成立”的初始实现使用方向结果进行结构化标记；未来接入更细的信号验证时，可按热点、资金、突破、均线和量价分别判断。

## 4. 汇总指标

- `accuracy.csv`：方向判断准确率。
- `win_rate.csv`：达到正式策略胜率收益阈值的比例。
- `return_tracking.csv`：每一条历史观察记录的完整结果。
- `signal_effectiveness.csv`：按信号、策略版本、观察周期统计样本数、正收益率、胜率、平均收益和平均最大回撤。

指标脚本每次都从 `runs/` 全量重算并覆盖 CSV，保证重复运行幂等，不重复累计。

## 5. 成功、失败与偏差

成功：

- 周期方向为正；
- 达到胜率收益阈值；
- 在有基准数据时跑赢指数或板块；
- 未触发风险条件。

失败：

- 周期方向为负；
- 最大回撤超出策略阈值；
- 入选信号在足够样本中持续低效；
- 热点成立但个股相对强度不足。

偏差：

- 幸存者偏差：只保留成功案例；
- 数据缺失偏差：把缺失行情视为失败；
- 回看偏差：用未来信息修改过去理由；
- 版本混淆：不同策略版本混在一起评价；
- mock 偏差：用模拟数据判断真实策略优劣。

## 6. 复盘纪律

- 保留原始 run 文件，不回写过去的选择结果。
- 汇总时必须带 `strategy_version`。
- 样本不足时明确写“暂不调整”。
- 先记录问题，再提出草案；不得直接覆盖正式策略。
